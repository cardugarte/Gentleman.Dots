# Revisión de Código

Realiza una revisión exhaustiva del código especificado en $ARGUMENTS

## Criterios de Revisión:

### 1. **Calidad del Código**
- Legibilidad y claridad
- Nombres descriptivos de variables y funciones
- Estructura y organización
- Complejidad ciclomática
- Comentarios y documentación

### 2. **Rendimiento**
- Optimizaciones posibles
- Uso eficiente de recursos
- Operaciones costosas
- Caching y memoización

### 3. **Seguridad**
- Validación de entradas
- Manejo de errores
- Vulnerabilidades conocidas
- Exposición de datos sensibles
- Inyección de código

### 4. **Mantenibilidad**
- Acoplamiento y cohesión
- Principios SOLID
- Design patterns apropiados
- Facilidad de testing

### 5. **Estilo y Convenciones**
- Consistencia con el resto del proyecto
- Convenciones de nomenclatura
- Formato y spacing
- Uso de linters

## Formato de Respuesta:

### Resumen
- Calificación general (1-10)
- Puntos fuertes
- Áreas de mejora

### Problemas Encontrados
Para cada problema:
- **Severidad**: Crítico / Alto / Medio / Bajo
- **Ubicación**: Archivo y línea
- **Descripción**: Qué está mal
- **Solución**: Cómo corregirlo
- **Código sugerido**: Ejemplo de implementación correcta

### Recomendaciones
- Mejoras generales
- Refactorizaciones sugeridas
- Próximos pasos
